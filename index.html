FIRST MISSION DESIGN YOUR SPRITE

拼豆精灵 = 用户养成的、由像素豆子构成的虚拟伙伴

先养成一个精灵，再开始你的拼豆宇宙。

新用户的第一个任务不是上传照片，而是在空白画布上拼出自己的精灵。精灵本身就是一张原创图纸，它会陪你生成、编辑、制作、分享，并随着创作成长。

看精灵机制 看创作流程
ORIGINAL FIRST BEAD PARTICLES AI CO-CREATION HEALING COMPANION SMART BOARD READY UGC CHALLENGE

01 / Sprite system

精灵不是小游戏，是核心创作机制本身。

FIRST TASK

空白画布设计精灵

新手从基础骨架开始改色、换表情、加配饰，第一张图纸 100% 原创，也天然规避版权风险。

GROW

随创作成长

完成图纸、打卡、参加挑战会解锁姿态、场景和配饰，但走治愈路线，不做焦虑式催促。

ASSIST

嵌入每个流程

上传时它观察，生成时它思考，编辑时给建议，完成时庆祝。陪伴感进入工具链路。

SHARE

分享自带识别点

晒图时带上自己的精灵形象，平台逐渐沉淀原创 IP，而不是只搬运外部 IP。

02 / Studio

从“图转图”升级到“万物转图纸”。

文字描述生成戴墨镜的橘猫，赛博朋克风
多图融合我 + 宠物 + 风格参考
指令式定制颜色少一点，更适合新手
空白画布从零拼出原创精灵

拼豆专属规则层

  • 颜色数量约束
  • 像素网格规则
  • 可制作性校验
  • 材料清单同步

壁垒不在通用生成模型，而在模型是否真的懂拼豆。

03 / Long memory

把推荐算法包装成“懂你的伙伴”。

记住题材 宠物、情侣、节日、国风
记住审美 常用色系、难度、尺寸
给出建议 “你上次很喜欢治愈系猫咪。”
情绪价值 陪伴，不催促；鼓励，不焦虑。

04 / UGC co-create

比晒成品更容易裂变的是接龙共创。

A

接力拼图

一张大图拆成多个区块，邀请好友或社群成员分别完成，最后合成完整作品。

B

同图二创挑战

同一底图，不同用户做不同配色，形成“一题多解”的内容池。

C

精灵带图出圈

分享时带上自己的精灵，让每张作品都有平台识别点。

05 / Smart board

图纸、精灵和智能拼豆板天然联动。

连接

APP 同步图纸、精灵状态和制作步骤。

引导

板子按颜色或区域发光，新手不用一直数格子。

庆祝

完成作品后精灵反馈，用户自然进入晒图和材料复购。

06 / MVP scope

第一版做小，但把“精灵”做深。

必须做

自定义精灵外形

基础骨架、改色、表情、保存为第一张原创图纸。

必须做

基础成长等级

完成图纸和创作行为带来轻量成长，不做重运营。

必须做

关键节点反馈

生成、编辑、完成时出现精灵反馈，增强陪伴感。

先留接口

多模态与共创

文字、多图、接力拼图、二创挑战先预留数据结构。
